# Development Notes & Process Documentation

This folder contains **internal development documentation** - notes, summaries, and process guides created during the project development.

## Purpose

These files document the **journey** of building this project:
- Refactoring decisions and before/after comparisons
- Feature implementation guides
- Deployment checklists for specific features
- Debugging notes and diagnostic reports
- Code quality improvement summaries

## Organization

### Implementation Summaries
- `*_IMPLEMENTATION.md` - Detailed feature implementation notes
- `*_SETUP.md` - Service setup and configuration guides
- `DEPLOY_*.md` - Feature-specific deployment procedures

### Before/After Documentation
- `*_SUMMARY.md` - Quick summaries of major changes
- `*_COMPLETE.md` - Comprehensive completion reports with metrics
- `DEPENDENCIES_BEFORE_AFTER.md` - Dependency pinning comparison

### Debugging & Diagnostics
- `DEBUG_VPS.md` - VPS debugging session notes
- `DIAGNOSTIC_REPORT.md` - System diagnostic outputs
- `QUICK_FIX.txt` - Quick fix notes

### Feature Guides
- `ATLANTIC_TRACKER_GUIDE.md` - Atlantic coverage implementation
- `FILTER_PANEL_UPGRADE.md` - Map filter panel feature
- `GEMINI_*.md` - AI intelligence scraper setup
- `GT_*.md` - Gross tonnage scraper implementations

### Deployment Checklists
